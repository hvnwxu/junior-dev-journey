## 🎯 목표

* **부분 점수(Subtask) 마인드셋 확립**: 만점이 아닌, 내가 풀 수 있는 조건의 점수를 확실히 따내는 훈련
* **1차원 리스트 값 비교 및 조건 탐색**: 리스트 내에서 이웃한 원소 간의 관계(`A[i-1]`과 `A[i]`)를 비교하는 감각 체화
* **정렬 여부 판단(Is-Sorted) 로직 마스터**: 데이터가 올바른 순서로 정렬되어 있는지 검사하는 흐름 이해

## ⏰ 하루 1.5시간 (90분) 코딩 루틴

1. **[15분] 웜업 (기계적 타이핑):** - 리스트 정렬 상태를 확인하는 기초 플래그 변수 패턴 코드 3회 타이핑
```python
is_sorted = True
for i in range(1, len(A)):
    if A[i-1] > A[i]:
        is_sorted = False

```


2. **[20분] 종이 로직 설계 (키보드 금지):** - 줄노트에 문제의 입력과 예제를 적고, 어떻게 칸을 이동하며 비교할지 손으로 화살표를 그리며 한글 순서 작성
3. **[40분] 코드 구현 및 디버깅:** - 종이에 적은 로직을 코드로 변환. 중간에 값이 어떻게 바뀌는지 `print()`를 찍어서 확인
4. **[15분] 깃허브 제출:** - 당일 작성한 코드를 업로드. 해결하지 못한 경우 막힌 원인을 주석으로 기록
