## 📦 1. 기초 체력 다지기: 자료구조 '리스트(List)'란?

코딩을 처음 할 때 변수를 '포스트잇'에 비유합니다. `a = 10`이라고 하면 `a`라는 포스트잇에 10을 적어두는 것이죠. 하지만 학생이 3,000명, 500,000명이 되면 포스트잇을 50만 개 붙일 수는 없습니다. 이때 사용하는 것이 리스트(List)입니다.

### 🏢 리스트는 '사물함 한 줄'이다

리스트는 하나의 이름 아래에 여러 개의 칸이 연속으로 붙어 있는 '번호가 매겨진 사물함'과 같습니다.

* **인덱스(Index)의 비밀 (0부터 시작):** 가장 중요한 규칙입니다. 컴퓨터는 숫자를 0부터 십니다. 사물함이 5칸짜리라면 번호는 **0, 1, 2, 3, 4**가 됩니다.
* **대응 관계:** 1번 학생의 정보는 `[0]`번 칸에, 2번 학생의 정보는 `[1]`번 칸에 저장하는 방식으로 다루어야 합니다.

### 이번 문제에서 리스트를 쓴 이유

학생들의 '집 위치'와 '학교 번호'를 따로따로 사물함에 순서대로 넣기 위해서입니다.

* `X`라는 사물함 리스트: 집 좌표들을 순서대로 저장
* `S`라는 사물함 리스트: 학교 번호들을 순서대로 저장
이렇게 하면 `X[0]`(0번 칸)과 `S[0]`(0번 칸)을 매칭하여 '1번 학생의 집은 X[0]이고, 학교는 S[0]이다'라고 한 번에 묶어서 관리할 수 있습니다.

## 🕵️‍♂️ 2. 1번 문제 '친구' 상세 해석 (손으로 풀기)

컴퓨터에게 일을 시키기 전에, 사람의 머리로 먼저 규칙을 완벽히 이해해야 합니다. 예제 1번 데이터를 직접 표로 그려서 조카에게 보여주세요.

* **입력 조건:** 학생 수 $N = 7$, 같은 학교 합격 거리 $K_1 = 3$, 다른 학교 합격 거리 $K_2 = 5$

| 학생 번호 | 코드에서의 위치 (인덱스) | 집 좌표 ($X$) | 학교 번호 ($S$) |
| --- | --- | --- | --- |
| **1번 학생** | `0` | **9** | **2** |
| **2번 학생** | `1` | **1** | **1** |
| **3번 학생** | `2` | **14** | **3** |
| **4번 학생** | `3` | **6** | **2** |
| **5번 학생** | `4` | **17** | **3** |
| **6번 학생** | `5` | **4** | **1** |
| **7번 학생** | `6` | **8** | **1** |

### 🏃‍♂️ 1번 학생(인덱스 0)의 입장이 되어보기

1번 학생(집 위치 9, 학교 2)이 사물함을 돌면서 누가 자기 친구인지 검사하는 과정입니다.

1. **2번 학생(`인덱스 1`) 검사:**
* 학교가 다릅니다 (1번은 학교 2, 2번은 학교 1). $\rightarrow$ 다른 학교 기준인 **거리 5 이하**여야 친구입니다.
* 실제 거리는 $|9 - 1| = 8$입니다. 5보다 크므로 **친구 아님!**


2. **4번 학생(`인덱스 3`) 검사:**
* 학교가 같습니다 (둘 다 학교 2). $\rightarrow$ 같은 학교 기준인 **거리 3 이하**여야 친구입니다.
* 실제 거리는 $|9 - 6| = 3$입니다. 3 이하에 딱 걸치므로 **친구 인정! (1명 확보)**


3. **7번 학생(`인덱스 6`) 검사:**
* 학교가 다릅니다 (1번은 학교 2, 7번은 학교 1). $\rightarrow$ 다른 학교 기준인 **거리 5 이하**여야 친구입니다.
* 실제 거리는 $|9 - 8| = 1$입니다. 5보다 작으므로 **친구 인정! (2명 확보)**



이런 식으로 **한 명이 나머지 모든 사람을 한 번씩 찔러보며 검사하는 방식**을 코드로 구현하는 것입니다.

## 💻 3. 코드 라인별 돋보기 설명

앞서 작성한 20점짜리 완전 탐색 코드를 한 줄씩 뜯어보겠습니다.

```python
import sys

```

* **설명:** 파이썬의 기본 기능 외에, 시스템(sys)과 관련된 강력한 도구 상자를 가져옵니다.

```python
N, K1, K2 = map(int, sys.stdin.readline().split())

```

* **`sys.stdin.readline()`:** `input()`과 똑같이 키보드 입력을 받는 명령어지만, 올림피아드처럼 대량의 데이터를 받을 때 백만 배 빠릅니다. 대회 필수 문법입니다.
* **`.split()`:** 들어온 한 줄의 글자를 공백(띄어쓰기) 기준으로 쪼갭니다. 예: `"7 3 5"` $\rightarrow$ `['7', '3', '5']`
* **`map(int, ...)`:** 쪼개진 글자들을 숫자로 형변환(글자 '7'을 진짜 숫자 7로) 시켜서 `N`, `K1`, `K2` 변수에 각각 매칭해 줍니다.

```python
X = []
S = []
for _ in range(N):
    x, s = map(int, sys.stdin.readline().split())
    X.append(x)
    S.append(s)

```

* **`X = []`, `S = []`:** 빈 사물함을 개설합니다.
* **`for _ in range(N):`:** 학생 수($N$번)만큼 아래 행동을 반복합니다. (변수 이름 자리에 `_`를 쓴 것은 횟수만 채우고 변수 자체는 안 쓰겠다는 뜻입니다.)
* **`.append(x)`:** 입력받은 집 좌표 `x`를 사물함 `X`에 차곡차곡 뒤로 이어 붙입니다.

```python
answer = [0] * N

```

* **설명:** 만약 $N=7$이라면 `[0, 0, 0, 0, 0, 0, 0]`이라는 리스트가 만들어집니다. 각 학생의 친구 숫자를 세어서 기록해 둘 '점수판 리스트'를 미리 세팅해 두는 것입니다.

```python
for i in range(N):
    count = 0
    for j in range(N):

```

* **2중 for문의 작동 원리 (핵심):**
* `i`는 '기준 학생'입니다. `i`가 0일 때(1번 학생), 안쪽 루프의 `j`는 0부터 끝까지 돌며 '비교 대상 학생'이 됩니다.
* 즉, `i=0`일 때 `j=0, 1, 2, 3, 4, 5, 6` 스캔을 쭉 하고, 끝나면 `i=1`이 되어 다시 `j=0, 1, 2, 3, 4, 5, 6` 스캔을 합니다.



```python
        if i == j:
            continue

```

* **설명:** 만약 `i`가 0이고 `j`가 0이면 자기 자신을 비교하는 상황입니다. 자기 자신과 거리를 재면 무조건 0이 나와서 친구로 카운트되므로, `continue`를 통해 아래 코드를 무시하고 다음 `j`로 넘겨버리는 방어벽입니다.

```python
        distance = abs(X[i] - X[j])

```

* **`abs()` 함수:** 절댓값(Absolute Value)을 구합니다. `X[i] - X[j]`를 했을 때 마이너스 값이 나오더라도 거리는 항상 양수여야 하므로 마이너스를 떼어버리는 역할을 합니다.

```python
        if S[i] == S[j] and distance <= K1:
            count += 1
        elif S[i] != S[j] and distance <= K2:
            count += 1

```

* **`if S[i] == S[j] and distance <= K1`:** 두 학생의 학교 사물함 값이 같고(`==`), 거리가 `K1` 이하이면 `count` 점수를 1점 올립니다.
* **`elif S[i] != S[j] and distance <= K2`:** 학교 사물함 값이 다르고(`!=`), 거리가 `K2` 이하이면 역시 `count` 점수를 1점 올립니다.

```python
    answer[i] = count

```

* **설명:** 안쪽 `j` 루프가 다 돌아서 한 명에 대한 검사가 끝나면, 최종 정산된 `count` 값을 점수판 리스트 `answer`의 `[i]`번 방에 저장합니다.

```python
print(*(answer))

```

* **설명:** 리스트 이름 앞에 별표(`*`)를 붙여서 출력하면 `[4, 2, 2]` 모양이 아니라 괄호가 다 벗겨진 `4 2 2` 형태로 출력됩니다. 올림피아드 정답 양식에 맞추기 위한 팁입니다.
